# Changelog

All notable changes to this project are documented here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- **`code_chunks` table creation** in `db/schema.sql` (pgvector `VECTOR(3072)`),
  so the vector store is provisioned automatically instead of manually.
- **Mermaid source export** in the chat "Execution flow" panel ‚Äî the raw diagram
  is shown alongside the text view so it can be pasted into a real renderer.
- Project docs: `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`,
  `ROADMAP.md`, `CHANGELOG.md`, `LICENSE` (MIT), and README terminal screenshots
  generated by `scripts/generate_screenshots.py`.

### Changed
- **Default embedding model** is now `models/gemini-embedding-001` (3072-dim) to
  match the `code_chunks` schema out of the box.
- **Grounding prompt** for chat: cite the chunk's real line range verbatim and
  base explanations on implementation code rather than docstrings/comments.

### Fixed
- **Duplicated chat output** ‚Äî long, streamed answers were re-drawn 5‚Äì20√ó in the
  terminal. Streaming now shows a spinner and renders the completed answer once.
- **Malformed Markdown rendering** ‚Äî over-indented list content and unbalanced
  code fences leaked raw `**` and ``` ``` ``` characters; output is now normalized
  before rendering.
- **Malformed Mermaid flow labels** ‚Äî node labels containing parentheses (e.g.
  `(blast radius)`) were truncated; bracket-aware parsing fixes this, and `&`
  joins render as "and".
- Removed stray debug `print()`s that corrupted the streamed CLI output.
- Schema `init_db()` no longer trips over semicolons inside SQL comments.

## [0.1.0] - 2026-07-01

Initial feature-complete milestone.

### Added
- **RAG ingestion pipeline** ‚Äî function-aware chunking (`services/chunker.py`),
  Gemini embeddings with retry/caching (`services/embeddings.py`), and pgvector
  storage with checksum-based incremental re-indexing (`services/embedder.py`).
- **Semantic retrieval** with MMR diversification (`services/retriever.py`).
- **Chat feature** ‚Äî streaming, multi-hop codebase Q&A with conversation memory,
  `@file` / `#symbol` reference scoping, source citations, and Mermaid flow
  diagrams (`agent/chat_agent.py`).
- **Multi-agent review pipeline** ‚Äî `context_agent` ‚Üí `review_agent` ‚Üí
  `retry_agent` orchestrated by `agent/orchestrator.py`, producing a structured
  `PRReview` with category, risk score, and confidence.
- **GitHub App webhook** ‚Äî signature-verified PR events trigger automated reviews
  posted back as comments (`routers/webhook.py`), with `USE_RAG_CONTEXT` /
  `USE_AGENT_REVIEW` / `AGENT_MODE` feature flags.
- **Typer CLI** ‚Äî `ingest`, `chat`, and `review` commands (`cli.py`).
- **Persistence** ‚Äî reviews, comments, agent traces, and chat sessions/messages
  in Postgres (`db/schema.sql`).
